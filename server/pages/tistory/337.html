블로그에 개인도메인을 연결하려는데...

2012. 1. 12. 00:59
블로그를 만드려면 가장 먼저 생각하는 것이 주소다. (주제가 물론 가장 중요하지만)
대부분의 서비스형 블로그에서 id만 만들면 되는데 좋은 id는 대부분 선점되어 있다. 그래서 생각한 것이 도메인을 구입해서 연결하는 것. (물론 도메인 주소도 좋은 건 다 선점되어 있긴하지만.. ^^;)

어쨌든 결국 도메인을 구매했고, 도메인을 연결해서 사용할 블로그는 어떤 걸 사용해야할까?
도메인을 포워딩 용도로 사용하는 것 말고 실제 블로그 주소를 내 도메인으로 만들어주는 것 위주로 조사를 해보았다.



티스토리

- URL : id.tistory.com

강점

  • 다양한 플러그인 기능을 사용
  • 강력한 스킨기능 (HTML, CSS를 거의 대부분 마음대로 조작이 가능)
  • 파비콘 변경가능
  • 도메인 연결기능 제공
  • 여러개의 블로그 운영가능
  • 팀 블로그 운영 가능

약점

  • 모바일 페이지에서는 강점 중 대부분이 사라진다. -_-
    플러그인 사용불가, 스킨은 아주 제한적으로 제공, URL이 바뀜. 그것도 tistory.com으로.. (가장 치명적임)



이글루스

- URL : id.egloos.com

강점

  • 강력한 스킨기능 (HTML, CSS 마음대로 조작가능)
  • 파비콘 변경가능
  • 도메인 연결기능 제공

약점

  • 모바일 페이지에서 URL이 일부변경된다. (치명적임)
  • 1인 1블로그만 운영가능



네이버 블로그

- URL : blog.naver.com/id, id.blog.me

강점

  • 네이버 특유의 깔금한 관리툴과 깔끔한 스킨 제공
  • 도메인 연결기능 제공
  • 네이버 검색에 매우 잘 노출된다.

약점

  • 스킨이 제한적 (네이버 블로그 티가 너무 난다.)
  • 파비콘 변경불가
  • 1인 1블로그만 운영가능
  • 팀블로그 운영불가
  • 모바일 페이지에서 URL이 완전히 변경된다. (굉장히 복잡해짐)



Tumblr

- URL : id.tumblr.com

강점

  • 트위터와의 연계가 강해서 긴글 트위터 기능으로 활용이 쉽다.
  • 강력한 스킨기능 (HTML, CSS 조작가능)
  • 파비콘 변경가능
  • 도메인 연결기능제공
  • 여러개의 블로그 운영가능
  • 모바일 페이지로 연결되도 URL이 변경되지 않는다.
  • custom url 생성가능

약점

  • 팀블로그 운영 불가 (정정 : 두번째 블로그부터는 팀블로그 운영가능)
  • 에디터가 조금 약한 느낌.



Blogger

- URL : id.blogspot.com

강점

  • 강력한 스킨기능 (HTML, CSS 조작가능)
  • 파비콘 변경가능
  • 도메인 연결기능제공
  • 여러개의 블로그 운영가능
  • 팀블로그 운영가능
  • 모바일 페이지로 연결되도 URL이 변경되지 않는다.

약점

  • 블로그 상단에 블로그 바가 항상 노출



Wordpress

- URL : id.wordpress.com

강점

  • 강력한 스킨기능 (HTML, CSS 조작가능)
  • 파비콘 변경가능
  • 도메인 연결기능제공
  • 여러개의 블로그 운영가능
  • 팀블로그 운영가능
  • 모바일 페이지로 연결되도 URL이 변경되지 않는다.
  • custom url 생성가능

약점

  • 대부분의 강력한 기능은 유료
    (HTML/CSS 편집, 파비콘, 도메인)




정리해보면 Wordpress가 가장 좋은데 유료라는... -_-;;
혼자 사용할 거라면 Tumblr 강력추천. 티스토리는 URL 문제때문에 아쉬워서 약간 추천 정도.
어쨌거나 다들 조금씩 장단이 있네. 완벽한 건 없구나.


돈 좀 쓸거라면 Wordpress를 쓰거나 호스팅을 받아서 설치형 블로그를 사용하는 것도 좋을 것 같다.
도메인 연결을 사용하지 않을 거라면 여기 언급하지 않은 블로그들도 다들 좋고..

끄읕!
